# Client Onboarding Checklist

## Gate 1 — Commercial acceptance

- [ ] Proposal accepted.
- [ ] Scope, fee and payment timing confirmed.
- [ ] Setup payment received or approved under an agreed exception.
- [ ] Client owner or manager named.
- [ ] TAD operator named.
- [ ] Pilot or service dates confirmed.

Do not create the live workspace before this gate is complete.

## Gate 2 — Authority and data boundary

- [ ] Client selection score is at least 12/16.
- [ ] Data authority confirmed in writing.
- [ ] Permitted data types reviewed.
- [ ] Sensitive and unnecessary information excluded.
- [ ] Record-removal and correction contact named.
- [ ] External messaging boundary explained.
- [ ] Approval responsibility accepted.

Do not import records before this gate is complete.

## Gate 3 — Baseline

- [ ] Active record count captured.
- [ ] Unowned records counted.
- [ ] Records without next actions counted.
- [ ] Records without due dates counted.
- [ ] Overdue follow-ups counted.
- [ ] First-response time measured where possible.
- [ ] Open quote age measured.
- [ ] Current weekly admin time estimated and source recorded.
- [ ] Baseline scorecard saved.

## Gate 4 — Workspace installation

- [ ] Managed business created.
- [ ] Correct client owner/manager membership added.
- [ ] TAD operator access confirmed.
- [ ] Sales Admin engagement installed.
- [ ] Workflow statuses checked against the client process.
- [ ] Closed statuses confirmed.
- [ ] Approval rules confirmed.
- [ ] Client can sign in to the Service Desk.
- [ ] Operator can access the client workspace.

## Gate 5 — Record preparation

- [ ] Source files copied to the protected working location.
- [ ] Import template used.
- [ ] Duplicate external references resolved.
- [ ] Dates normalised to `YYYY-MM-DD`.
- [ ] Quote values contain numbers only.
- [ ] Closed records separated from active records.
- [ ] Notes reduced to operational facts.
- [ ] At least one authorised contact method exists where follow-up is required.
- [ ] Import reviewed with the client.

## Gate 6 — Launch

- [ ] At least 10 real records loaded.
- [ ] Every active record has a current status.
- [ ] At least 80% have owner, next action and due date by day 3.
- [ ] First operator queue review completed.
- [ ] First real outcomes recorded.
- [ ] Approval route tested with a real or client-approved low-risk decision.
- [ ] Week 1 review booked.
- [ ] Week 2 closeout booked.

## Gate 7 — Operating rhythm

Every business day:

- [ ] Review due and overdue actions.
- [ ] Complete authorised follow-up work.
- [ ] Record the real outcome.
- [ ] Set the next action and due date.
- [ ] Escalate blockers.
- [ ] Route material decisions to the Service Desk.
- [ ] Record operator time and work performed outside DueToday.

## Gate 8 — Closeout

- [ ] Two weekly reports generated.
- [ ] Client reviewed both reports.
- [ ] Continue/change/stop response recorded.
- [ ] Endline scorecard completed.
- [ ] Operator time totalled.
- [ ] Exceptions ranked.
- [ ] Willingness-to-pay answer recorded.
- [ ] Continue, change or stop commercial action agreed.
